# UI/UX Review Checklist

## Workflow Clarity
- [ ] The primary task on each screen is obvious.
- [ ] Navigation reflects role boundaries and user intent.
- [ ] The sequence of steps is clear for multi-step workflows.
- [ ] System state such as archived, inactive, deleted, or locked is visible and understandable.

## Forms and Validation
- [ ] Labels are explicit and user-facing.
- [ ] Required fields and constraints are visible before submission where practical.
- [ ] Validation appears inline and explains how to recover.
- [ ] Destructive or blocked actions give clear reason and next step.

## Empty, Error, and Edge States
- [ ] Empty states include short explanatory text and a primary action where appropriate.
- [ ] Error states are understandable and actionable.
- [ ] Long or restricted inputs provide counters, limits, or helper text when needed.
- [ ] Mention, tag, and status interactions define failure behavior clearly.

## Fluent UI Consistency
- [ ] Layout uses clear hierarchy and consistent spacing.
- [ ] Commands, dialogs, panels, and lists follow Fluent UI-style interaction patterns.
- [ ] Interactive controls are large enough and predictable.
- [ ] Administrative screens and collaboration screens feel distinct but consistent.

## Accessibility and Readability
- [ ] Information density remains readable for frequent-use screens.
- [ ] Text hierarchy is strong enough for scanning.
- [ ] Keyboard-friendly behavior is considered for primary actions.
- [ ] Important feedback is not communicated by color alone.
