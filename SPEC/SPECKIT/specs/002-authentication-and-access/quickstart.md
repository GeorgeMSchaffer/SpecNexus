# Quickstart: Authentication and Access

1. Start the system with a valid environment-provided Site Admin credential.
2. Verify the seeded Site Admin can log in and is forced to change password.
3. Verify invalid credentials return the expected non-2xx error shape.
4. Verify inactive users are denied.
5. Verify 5 failed attempts in 15 minutes lead to a 15-minute lockout.
6. Verify email uniqueness is enforced across the system.
